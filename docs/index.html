Simulating Advanced Drone Networks

A robust platform for modeling drone communication, latency, packet loss, and more.

Key Features

Dynamic Network Simulation

Simulate real-time network conditions such as latency, jitter, and packet loss to create a realistic communication environment for your drone or robotic agent. These network factors are essential for understanding how communication delays and disruptions can impact performance.

  • Latency Simulation: Introduces time delays to replicate network transmission times, which can be adjusted based on real-world network conditions.
  • Jitter Simulation: Adds random fluctuations to the timing of packet arrivals, mimicking real-world network instability and its effect on control systems.
  • Packet Loss Simulation: Randomly drops packets to simulate network outages or congestion, testing how the system handles missing or corrupted data.
  • Throughput Metrics: Monitors data transfer rates and ensures that the system can handle variable bandwidth scenarios effectively.

ZeroMQ Integration

Leverage ZeroMQ for efficient, low-latency, and fault-tolerant communication between the simulation environment and the drone. ZeroMQ supports multiple messaging patterns (e.g., Publisher-Subscriber, Request-Reply) and handles real-time data transmission in a scalable manner.

  • Publisher-Subscriber Model: Send sensor data and control commands from the environment (or ground control station) to the drone, supporting real-time updates.
  • Request-Reply Model: Allows for two-way communication where the drone sends responses (e.g., state updates or acknowledgment) back to the controller, ensuring synchronization and feedback loops.
  • Message Serialization: Data (e.g., control actions, observations) is serialized (typically in JSON format) before transmission, ensuring that complex structures are handled correctly over the network.
  • Low Latency and High Throughput: ZeroMQ optimizes communication to minimize delays and maximize data transfer speeds, crucial for time-sensitive applications like autonomous drones.

Customizable Environments

Adjust simulation parameters to create a wide range of drone scenarios, testing everything from basic navigation to complex multi-agent interactions under various environmental conditions.

  • Target Generation: Dynamically generate targets for the drone to reach, simulating navigation tasks in a variety of terrains and environments.
  • Real-Time Data Feedback: Monitor and adjust parameters like altitude, position, velocity, and environmental conditions during the simulation, allowing for a more accurate testing environment.
  • Simulated Obstacles and Hazards: Introduce dynamic obstacles (e.g., other drones, terrain features) to test how the drone reacts to unforeseen challenges.
  • Adjustable Network Conditions: Fine-tune the network settings (latency, jitter, packet loss) to simulate different levels of network reliability, providing insight into how the drone adapts to varying connectivity conditions.

How to Install AeroLinkSim

  1. Clone the repository: git clone https://github.com/Nakok/AeroLinkSim.git
  2. Navigate to the project directory: cd AeroLinkSim
  3. Install dependencies: pip install -r requirements.txt
  4. Run the environment: python run.py

Usage

Use the following code to start simulating drone networks:

      
from agent import PPOAgent
from env import AeroLinkSimEnv

env = AeroLinkSimEnv()
agent = PPOAgent(env)

for episode in range(100):
    agent.train()
    print(f"Episode {episode + 1} completed.")
